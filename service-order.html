주문 및 장바구니

  1. 홈
  2. 주문 및 장바구니

상품 선택부터 주문 완료까지 끊기지 않는 흐름을 구현했습니다

쇼핑몰 프로젝트에서 상품 조회, 장바구니, 수량 변경, 주문서 작성, 주문 상태 관리까지 전체 주문 플로우를 직접 구현했습니다. 사용자 화면과 관리자 확인 흐름이 연결되도록 데이터 구조와 화면 동작을 함께 맞췄습니다.

핵심 구현 내용

단일 기능이 아닌 주문 라이프사이클 중심으로 정리했습니다. 장바구니 변경 내용이 주문서와 DB에 정확히 반영되도록 처리했고, 중간 이탈이나 비정상 입력 같은 예외 상황도 함께 검증했습니다.

상품 선택 및 상세 조회

상품 목록/상세 화면에서 옵션과 수량을 선택하고 바로 장바구니 또는 주문으로 이어지도록 구성했습니다. 품절 상품이나 비정상 요청은 안내 메시지와 함께 차단했습니다.

장바구니 처리와 수량 변경

장바구니 담기, 수량 증가/감소, 선택 삭제 기능을 구현하고 총 금액이 즉시 반영되도록 처리했습니다. 수량이 0 이하로 내려가는 상황이나 중복 담기 케이스도 별도로 처리했습니다.

주문 처리 및 상태 관리

주문서 작성 후 주문 데이터를 저장하고, 주문 상태를 단계별로 관리했습니다. 사용자 주문 내역 조회와 관리자 주문 확인 흐름을 연결해 운영 화면에서도 동일한 상태를 확인할 수 있게 했습니다.

예외 케이스 대응

로그인 만료 상태에서 주문 요청, 잘못된 수량 입력, 이미 처리된 주문 재요청 같은 케이스를 검증해 중복 처리와 데이터 꼬임을 방지했습니다.

주문 기능 구현 포인트

01

주문 플로우 정의

상품 상세 → 장바구니 → 주문서 → 주문 완료 흐름을 먼저 고정했습니다.

02

상태 데이터 설계

장바구니 항목, 주문 테이블, 주문 상태 값의 관계를 명확히 정리했습니다.

03

화면/DB 동기화

수량 변경, 금액 계산, 주문 저장 결과가 화면과 DB에서 동일하게 반영되도록 구현했습니다.

04

예외 상황 점검

로그인 만료, 비정상 수량, 중복 요청 상황을 테스트하며 처리 로직을 보완했습니다.

주 사용 기술

  • 백엔드: PHP, Laravel, Spring
  • 프론트엔드: HTML, CSS, Bootstrap, JavaScript
  • 데이터베이스: MySQL, Oracle, MS-SQL
  • 도구: Git, GitHub, VS Code

구현 포인트

  • 장바구니: 담기/삭제/수량 변경/총액 재계산 처리
  • 주문서: 배송 정보와 주문 항목 검증 후 저장
  • 상태 관리: 주문 접수 이후 단계별 상태 값 관리
  • 관리자 연동: 주문 확인 화면과 사용자 내역 화면 동기화

관련 프로젝트 연결

  • 온라인 쇼핑몰: 상품, 장바구니, 주문 처리 핵심 기능 구현
  • 베이커리 관리 웹: 주문·판매 데이터 기반 운영 화면 경험
  • 트러블슈팅 기록: 주문 흐름 예외 처리 과정 블로그 정리

사용 기술

  • Laravel/PHP 기반 주문 로직과 상태 처리
  • JavaScript를 활용한 장바구니 수량 및 금액 반영
  • MySQL 연동으로 주문/장바구니 데이터 관리

바로 이동

주문 및 장바구니 흐름은 쇼핑몰 프로젝트 상세 페이지에서 화면 단위로 확인할 수 있습니다.
