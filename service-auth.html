회원 및 권한 관리

  2. 회원 및 권한 관리

회원가입부터 관리자 접근 제어까지 인증/인가 흐름을 직접 구현했습니다

쇼핑몰과 관리형 웹 프로젝트에서 회원가입, 로그인, 세션 유지, 권한 분기를 한 흐름으로 설계했습니다. 단순 로그인 화면이 아니라 사용자/관리자 경계를 분명히 두고 접근 제어를 구현한 경험을 정리했습니다.

핵심 구현 내용

기능 단위 설명이 아니라 실제 구현 기준으로 정리했습니다. 인증 상태 유지, 권한에 따른 메뉴/기능 노출, 관리자 페이지 접근 제한까지 운영 관점에서 필요한 항목을 함께 다뤘습니다.

회원가입/로그인 흐름

유효성 검증, 중복 계정 확인, 로그인 실패 처리까지 포함해 회원가입과 로그인 흐름을 구현했습니다. 입력값 누락이나 중복 데이터 같은 예외 케이스를 초기에 걸러 실제 사용 중 오류를 줄였습니다.

세션 처리 및 상태 유지

로그인 성공 시 세션에 사용자 정보를 저장하고, 필요한 페이지에서 세션 존재 여부를 점검해 접근을 제어했습니다. 로그아웃 시 세션 정리까지 포함해 인증 상태가 일관되게 동작하도록 구성했습니다.

권한 분기와 관리자 접근 제어

일반 사용자와 관리자 계정을 분리하고, 관리자 전용 기능은 권한 확인 로직을 통과한 경우에만 실행되도록 구현했습니다. URL 직접 접근 시에도 권한 검사를 적용해 우회 접근을 막았습니다.

문제 해결 경험

세션 만료 타이밍 차이로 발생하던 화면 오류를 재현해 확인하고, 공통 체크 로직을 정리해 페이지별 처리 편차를 줄였습니다. 로그인 상태와 화면 노출이 어긋나던 문제도 조건식 정리로 해결했습니다.

인증/인가 구현 포인트

01

요구사항 분리

회원 기능과 관리자 기능을 먼저 분리하고, 각 화면에서 필요한 권한 범위를 명확히 정의했습니다.

02

세션 기준 설계

로그인 유지 기준, 만료 처리, 로그아웃 동작을 먼저 설계해 상태 꼬임을 줄였습니다.

03

권한 체크 공통화

페이지 단위로 권한 확인 로직을 통일해 우회 접근과 중복 코드를 함께 줄였습니다.

04

예외 흐름 검증

로그인 실패, 세션 만료, 비정상 접근 시 메시지와 이동 경로를 점검하며 안정성을 보완했습니다.

주 사용 기술

  • Back-End: PHP, Laravel, Spring
  • Front-End: HTML, CSS, Bootstrap, JavaScript
  • Database: MySQL, Oracle, MS-SQL
  • Tools: Git, GitHub, VS Code

구현 포인트

  • 회원가입: 중복 검사, 필수값 검증, 실패 메시지 처리
  • 로그인/세션: 세션 저장, 유지, 로그아웃 정리
  • 권한 분기: 사용자/관리자 메뉴 및 기능 분리
  • 접근 제어: 관리자 URL 직접 접근 차단 로직 적용

관련 프로젝트 연결

  • 쇼핑몰 프로젝트: 회원가입, 로그인, 마이페이지 흐름 구현
  • 베이커리 관리 웹: 관리자 계정 기반 관리 메뉴 접근 제어
  • e스포츠 정보 포털: 사용자 중심 정보 화면과 권한별 기능 분리

사용 기술

  • PHP/Laravel 기반 인증 처리와 세션 관리
  • Spring/JSP 구조에서 권한 분기 로직 구성
  • HTML/CSS/Bootstrap/JavaScript로 로그인·마이페이지 UI 연동
