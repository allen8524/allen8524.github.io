DB 설계 및 연동

  1. 홈
  2. DB 설계 및 연동

기능 요구사항을 테이블 구조와 쿼리로 연결해 구현했습니다

회원, 주문, 게시글처럼 서로 연결되는 데이터를 기준으로 ERD를 정리하고 정규화한 뒤, 화면 기능과 DB 로직을 함께 구현했습니다. 단순 저장이 아니라 조회 흐름과 예외 처리까지 고려한 DB 연동 경험을 담았습니다.

핵심 구현 내용

데이터 모델링부터 쿼리 구성, 예외 처리까지 실제 구현 범위를 중심으로 정리했습니다. 화면 요구사항이 바뀌어도 테이블 관계와 쿼리 흐름이 유지되도록 설계하는 데 집중했습니다.

테이블 관계 설계와 정규화

회원, 상품, 주문, 게시글 테이블의 관계를 기준으로 ERD를 작성하고 중복 컬럼을 정리해 정규화했습니다. 기능 추가 시 확장 가능한 구조를 우선 고려했습니다.

CRUD 및 조회 쿼리 흐름

기본 CRUD뿐 아니라 목록/상세 화면에 필요한 JOIN 조회를 구현했습니다. 조건별 조회와 정렬 로직을 분리해 유지보수 시 수정 범위를 줄였습니다.

웹 기능과 DB 연동

컨트롤러에서 전달된 입력값이 DB에 반영되고, 다시 화면으로 정확히 조회되도록 데이터 흐름을 구성했습니다. Laravel, Spring 프로젝트에서 MySQL, Oracle, MS-SQL 연동 경험을 쌓았습니다.

예외 처리와 데이터 일관성

잘못된 파라미터, 누락 데이터, 중복 입력 같은 예외 케이스를 사전에 검증했습니다. 삭제/수정 시 연관 데이터 영향을 함께 확인해 무결성 문제가 발생하지 않도록 처리했습니다.

DB 설계/연동 구현 포인트

01

요구사항 기반 모델링

화면 기능을 기준으로 엔터티와 관계를 정의하고 ERD를 먼저 작성했습니다.

02

정규화 및 키 설계

중복 데이터를 줄이고 PK/FK 기준을 명확히 정리했습니다.

03

쿼리/연동 구현

CRUD와 JOIN 조회를 구현하고 컨트롤러-서비스-DB 흐름을 연결했습니다.

04

무결성 검증

예외 입력과 연관 데이터 변경 시나리오를 점검해 일관성을 보완했습니다.

주 사용 기술

  • Back-End: PHP, Laravel, Spring
  • Front-End: HTML, CSS, Bootstrap, JavaScript
  • Database: MySQL, Oracle, MS-SQL
  • Tools: Git, GitHub, VS Code

구현 포인트

  • 모델링: ERD 작성 및 테이블 관계 정의
  • 쿼리 구성: CRUD, JOIN, 조건 조회 로직 분리
  • 연동 처리: 화면 입력값과 DB 결과의 일관된 매핑
  • 예외 대응: 중복/누락/무결성 이슈 사전 검증

관련 프로젝트 연결

  • 온라인 쇼핑몰: 회원-상품-주문 관계 테이블 설계/연동
  • e스포츠 포털: 콘텐츠 데이터 구조와 조회 쿼리 구성
  • 베이커리 관리 웹: 판매 데이터 저장/집계용 DB 처리 경험

사용 기술

  • MySQL, Oracle, MS-SQL 기반 데이터 처리
  • Laravel/Spring 환경의 DB 연동 및 쿼리 작성
  • 정규화, 관계 설계, 무결성 검증 중심 접근

바로 이동

DB 설계와 연동 방식은 프로젝트 상세 페이지에서 기능 흐름과 함께 확인할 수 있습니다.
