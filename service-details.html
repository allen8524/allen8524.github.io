웹 서비스 구현 역량

  1. 홈
  2. 웹 서비스 구현 역량

회원, 주문, 게시판, DB까지 연결해서 구현해본 웹 개발 역량

쇼핑몰, 정보 포털, 관리형 웹 프로젝트를 진행하며 로그인부터 데이터 저장, 관리자 처리, 화면 연결까지 이어지는 흐름을 직접 구현해왔습니다. 이 페이지는 제가 실제로 구현 가능한 핵심 기능 범위를 정리한 포트폴리오 상세 페이지입니다.

구현 가능한 기능

아래 항목은 개념 소개가 아니라 실제 프로젝트에서 구현해본 기능을 기준으로 정리했습니다. 사용자 화면에서 끝나는 기능이 아니라 데이터 구조, 관리자 처리, 예외 흐름까지 함께 고려해 개발하는 쪽에 강점을 두고 있습니다.

회원 및 권한 관리

회원가입, 로그인, 세션 기반 인증, 일반 사용자와 관리자 권한 분리, 마이페이지 정보 조회 및 수정, 접근 제어 같은 기본 회원 기능을 구현할 수 있습니다. 필요한 경우 관리자 전용 화면과 일반 사용자 화면의 동작 범위도 분리해서 처리합니다.

주문 및 장바구니 기능

상품 목록 및 상세 조회, 장바구니 담기와 수량 변경, 주문서 작성, 주문 상태 관리, 관리자 측 주문 확인 흐름까지 연결해서 구현할 수 있습니다. 쇼핑몰 프로젝트에서는 주문 데이터가 실제로 어떻게 저장되고 관리되는지까지 직접 다뤘습니다.

게시판 및 콘텐츠 관리

게시글 작성, 수정, 삭제, 상세 조회, 검색, 페이징, 관리자 콘텐츠 관리 같은 전형적인 게시판 기능을 구현할 수 있습니다. 단순 CRUD를 넘어서 운영 화면에서 콘텐츠를 관리하는 흐름까지 엮어보는 경험을 쌓았습니다.

DB 설계 및 연동

회원, 상품, 주문, 게시글처럼 서로 연결되는 데이터를 기준으로 테이블 관계를 정리하고, CRUD와 JOIN 기반 조회를 구현할 수 있습니다. Laravel, Spring 기반 프로젝트에서 MySQL, Oracle, MS-SQL을 다뤄보며 데이터 구조와 화면 로직을 함께 맞춰왔습니다.

구현 접근 방식

01

요구사항 정리

기능 단위로 필요한 입력값, 예외 상황, 관리자 처리 여부를 먼저 정리하고 화면 흐름을 잡습니다.

02

테이블 및 권한 구조 설계

ERD와 테이블 관계를 정리하고, 사용자/관리자 권한 분리와 데이터 흐름을 먼저 설계합니다.

03

기능 구현 및 연동

컨트롤러, 서비스 로직, DB 처리, 화면 연결을 기능 단위로 구현하고 필요한 검증을 함께 넣습니다.

04

테스트 및 보완

정상 흐름뿐 아니라 예외 입력과 권한 분기까지 확인하면서 실제 사용에 맞게 동작을 다듬습니다.

주 사용 기술

  • 백엔드: PHP, Laravel, Spring
  • 프론트엔드: HTML, CSS, Bootstrap, JavaScript
  • 데이터베이스: MySQL, Oracle, MS-SQL
  • 도구: Git, GitHub, VS Code

경험한 기능 범위

  • 인증 흐름: 로그인, 회원가입, 권한 분리, 세션 처리
  • 쇼핑몰 기능: 상품 조회, 장바구니, 주문서, 주문 상태 관리
  • 운영 기능: 게시판 CRUD, 검색, 페이징, 관리자 화면
  • 데이터 처리: 테이블 설계, CRUD, JOIN, 화면 연동

학습 중인 기술

  • 백엔드 확장: Spring 구조 이해와 서비스 계층 설계 심화
  • DB 활용: 더 안정적인 쿼리 구성과 성능 관점의 접근
  • 협업 방식: Git 기반 작업 흐름과 코드 정리 습관 강화

개발 핵심 역량 요약

  • 기능 하나를 따로 구현하기보다 사용자 화면, 관리자 화면, 데이터 저장 구조를 연결해서 보는 편입니다.
  • 쇼핑몰과 관리형 웹 프로젝트를 통해 회원, 주문, 게시판, 관리자 기능을 실제로 구현해봤습니다.
  • 없는 경력을 꾸미기보다 직접 만들어본 기능 범위를 기반으로 확장해가는 개발 방향을 중요하게 생각합니다.
